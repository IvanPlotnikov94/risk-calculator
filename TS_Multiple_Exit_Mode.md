# Техническое задание: Реализация режима "Multiple Exit"

## 1. Общая информация

### 1.1 Цель задачи
Расширить функциональность приложения Risk Calculator, добавив новый режим работы "Multiple Exit" (множественные точки выхода), который является зеркальной противоположностью существующему режиму "Multiple Entry" (множественные точки входа).

### 1.2 Контекст
В текущей реализации приложение поддерживает только один режим: пользователь задает несколько точек входа (с разной ценой и объемом), один общий Stop Loss и один общий Take Profit. Новая функциональность позволит пользователю задать один вход, один Stop Loss и несколько точек выхода (Take Profit) с распределением объема позиции между ними.

### 1.3 Соглашения
- Все тексты интерфейса на русском языке
- Код, комментарии к коду, названия переменных и функций — на английском языке
- Числовые значения: поддержка как точки ".", так и запятой "," в качестве десятичного разделителя

- **Тип**: Числовое поле ввода
- **Формат**: С плавающей точкой, поддержка "." и ","
- **Валидация**:
  - Значение > 0

##### Столбец 2: "% объема позиции"
- **Тип**: Числовое поле ввода с суффиксом "%"
- **Формат**: Целое или дробное число
- **Placeholder**:
  - Первая строка: `1-100%`
  - Последующие строки: `1-{maxAvailablePercent}%`, где `maxAvailablePercent = 100 - Σ(% предыдущих строк)`
- **Валидация**:
  - Значение > 0
  - Значение ≤ maxAvailablePercent (остаток нераспределенного объема)
  - Только цифры и один десятичный разделитель
- **Сообщение об ошибке**: "Доступно для распределения: {maxAvailablePercent}%" (оптимизируй сообщение об ошибке и сделай наглядным предупреждением)

##### Столбец 3: "Средняя цена выхода"
- **Тип**: Рассчитываемое поле (только чтение)
- **Формат отображения**: `3049.00` (2 знака после запятой)
- **Формула**:
  ```
  avgExitPrice = Σ(exitPrice_i × percent_i) / Σ(percent_i)
  ```
  где percent_i — процент объема для i-го выхода (в долях от 1)

##### Столбец 4: "Объем выхода (USDT)"
- **Тип**: Рассчитываемое поле (только чтение)
- **Формат отображения**: `$100.00`
- **Формула**:
  ```
  volumeUSDT_i = totalVolume × (percent_i / 100)
  ```
  где totalVolume — "Объем позиции (USDT)"

##### Столбец 5: "Объем выхода"
- **Тип**: Рассчитываемое поле (только чтение)
- **Формат отображения**: `0.0328 BTC` (тикер подставляется из поля "Тикер")
- **Формула**:

##### Столбец 6: "% до TP"
- **Тип**: Рассчитываемое поле (только чтение)

- **Формула** (процент движения от Entry):
  - Для Long: `percentToTP_i = (exitPrice_i - entryPrice) / entryPrice × 100`
  - Для Short: `percentToTP_i = (entryPrice - exitPrice_i) / entryPrice × 100`

- **Тип**: Рассчитываемое поле (только чтение)

##### Столбец 8: "PnL при SL"
- **Тип**: Рассчитываемое поле (только чтение)
- **Формат отображения**: `-$6.67` (красный) или `$44.08` (зеленый)
- **Описание**: Прибыль/убыток при достижении текущей цены выхода, после чего цена идет к Stop Loss (оптимизируй это описание)

##### Столбец 9: "R/R"

### 5.3 Формулы расчета PnL

#### 5.3.1 Общие обозначения
- `entryPrice` — цена входа
- `stopLoss` — цена Stop Loss
- `exitPrices[]` — массив цен выхода (отсортированный по возрастанию для Long, по убыванию для Short)
- `percents[]` — массив процентов объема для каждого выхода
- `totalVolume` — общий объем позиции в USDT

#### 5.3.2 Расчет PnL при TP для i-го выхода
**Логика**: Цена доходит до i-го выхода, закрываются все выходы до i включительно, затем цена идет к средневзвешенной цене всех выходов (TP).

```
pnlAtTP_i = Σ(j=0 to i) [volumeUSDT_j × (exitPrice_j - entryPrice) / exitPrice_j]  // для Long
pnlAtTP_i = Σ(j=0 to i) [volumeUSDT_j × (entryPrice - exitPrice_j) / exitPrice_j]  // для Short
```

**Примечание**: Для расчета PnL при TP предполагается, что после закрытия всех выходов до i-го включительно, оставшаяся часть позиции закрывается на среднем TP всех выходов.

Альтернативная интерпретация (более точная):
- PnL при TP для i-го выхода = суммарная прибыль от всех закрытых выходов (от 0 до i)
- Это накопительная прибыль от частичных закрытий

```
pnlAtTP_i = Σ(j=0 to i) [volumeUSDT_j / exitPrice_j × (exitPrice_j - entryPrice)]  // для Long
```

где `volumeUSDT_j / exitPrice_j` — количество актива, закрытого на j-м выходе.

#### 5.3.3 Расчет PnL при SL для i-го выхода
**Логика**: Цена доходит до i-го выхода, закрываются все выходы до i включительно с прибылью, затем цена идет к Stop Loss и оставшаяся часть позиции закрывается с убытком.

```
pnlAtSL_i = pnlAtTP_i + (remainingVolume / stopLoss) × (stopLoss - entryPrice)  // для Long
pnlAtSL_i = pnlAtTP_i + (remainingVolume / stopLoss) × (entryPrice - stopLoss)  // для Short
```

где:
- `pnlAtTP_i` — прибыль от закрытых выходов (от 0 до i)
- `remainingVolume = totalVolume - Σ(j=0 to i) volumeUSDT_j` — оставшийся объем после закрытия первых i выходов

**Важно**: PnL при SL для i-го выхода ВСЕГДА > PnL при SL для (i-1)-го выхода, так как с каждым новым достигнутым выходом фиксируется больше прибыли.

#### 5.3.4 Пошаговый пример расчета

**Дано:**
- Direction: Long
- Entry: 88000
- SL: 85000
- Volume: $1000
- TP-1: 90000 (40%)
- TP-2: 91000 (25%)
- TP-3: 92000 (20%)
- TP-4: 94000 (15%)

**Расчеты для TP-1 (90000, 40%):**
- Объем закрытия: $1000 × 0.40 = $400
- Количество BTC: $400 / 90000 = 0.00444 BTC
- Прибыль на TP: 0.00444 × (90000 - 88000) = $8.89
- Оставшийся объем: $600
- Убыток на SL: ($600 / 85000) × (85000 - 88000) = -$21.18
- **PnL при TP-1**: $8.89
- **PnL при SL после TP-1**: $8.89 + (-$21.18) = -$12.29

**Расчеты для TP-2 (91000, 25%):**
- Объем закрытия: $1000 × 0.25 = $250
- Количество BTC: $250 / 91000 = 0.00275 BTC
- Прибыль на TP-2: 0.00275 × (91000 - 88000) = $8.24
- Накопленная прибыль (TP-1 + TP-2): $8.89 + $8.24 = $17.13
- Оставшийся объем: $350
- Убыток на SL: ($350 / 85000) × (85000 - 88000) = -$12.35
- **PnL при TP-2**: $17.13
- **PnL при SL после TP-2**: $17.13 + (-$12.35) = $4.78

**Проверка инвариантов:**
- PnL при SL для TP-2 ($4.78) > PnL при SL для TP-1 (-$12.29) ✓
- PnL при SL для TP-1 (-$12.29) > Общий риск при SL (без достижений TP) ✓

### 5.6 Блок "Сводка по позиции"

#### 5.6.1 Структура блока
Отображается под таблицей и содержит:

| # | Название | Формат | Описание |
|---|----------|--------|----------|
| 1 | Средняя цена | `3056.25` | Средневзвешенная цена всех выходов |
| 2 | Общий объем | `0.328 ETH` | Суммарный объем в активе |
| 3 | Общая сумма | `$1,000.00` | Суммарный объем в USDT |
| 4 | Риск (SL) | `-$30.00` | Убыток при достижении SL без закрытия на TP |
| 5 | Прибыль (TP) | `+$120.00` | Прибыль при достижении всех TP |
| 6 | R/R | `4.00` | Risk/Reward ratio при полном закрытии |

#### 5.6.2 Формулы расчета

##### Средняя цена выхода
```
avgExitPrice = Σ(exitPrice_i × percent_i) / Σ(percent_i)
```
где percent_i — в долях от 1.

##### Общий объем (в активе)
```
totalVolumeTicker = Σ(volumeUSDT_i / exitPrice_i)
```

##### Общая сумма (USDT)
```
totalVolumeUSDT = Σ(volumeUSDT_i)
```
Равна полю "Объем позиции" при корректном распределении 100%.

##### Риск (SL)
Убыток при недостижении ни одного TP:
```
riskSL = (totalVolume / stopLoss) × (stopLoss - entryPrice)  // для Long
riskSL = (totalVolume / stopLoss) × (entryPrice - stopLoss)  // для Short
```

##### Прибыль (TP)
Прибыль при достижении всех TP:
```
profitTP = Σ [volumeUSDT_i / exitPrice_i × (exitPrice_i - entryPrice)]  // для Long
profitTP = Σ [volumeUSDT_i / exitPrice_i × (entryPrice - exitPrice_i)]  // для Short
```

##### R/R
```
riskReward = |profitTP| / |riskSL|
```

**Важно**: R/R в сводке должен быть равен R/R последней строки таблицы (при достижении всех TP).
