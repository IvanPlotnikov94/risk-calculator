Memory & AI Rules

1. Контекст проекта
Это калькулятор риска для трейдинга (Risk Calculator).
Важнейший файл: Всегда обращайся к ARCHITECTURE.md за формулами расчетов, структурой папок и описанием компонентов.

2. Критически важная логика (Must Follow)
Эти правила предотвращают ошибки в расчетах. ИИ обязан соблюдать их при генерации кода.

Порядок исполнения входов (Execution Order)
При расчетах executionOrderEntries и partialScenarios критически важен порядок сортировки:

Long позиции: Входы сортируются по УБЫВАНИЮ цены (от высокой к низкой).
Short позиции: Входы сортируются по ВОЗРАСТАНИЮ цены (от низкой к высокой).Причина: Это имитирует реальный процесс набора позиции при движении цены.

Валидация данных

Расчет positionSummary (сводка по позиции) выполняется ТОЛЬКО по валидным входам.
Валидный вход: Цена находится между Stop Loss и Take Profit.
Невалидный вход: В расчетах не участвует, в UI отображаются только поля ввода.

3. Технические стандарты (Coding Conventions)
Соблюдай эти стандарты при написании кода:

State Management: Вся бизнес-логика и формулы должны находиться в stores/calculator.ts. Компоненты Vue (*.vue) должны быть "глупыми" — только отображение и вызов actions.
Styling: Используем Tailwind CSS. Избегай написания кастомных CSS-классов, используй утилитарные классы.
Validation: Для проверки входных данных используй библиотеку Zod.
TypeScript: Строгая типизация. Все типы и интерфейсы храни в types/index.ts.

4. Тестирование
Используем Vitest.
При добавлении новой логики расчетов (например, изменение формулы PnL) обязательно пиши unit-тест.

5. Предпочтения по ответам AI
Если пользователь просит изменить логику расчетов, сначала покажи формулу из ARCHITECTURE.md, которую планируешь изменить, и объясни последствия.
Не предлагай установку дополнительных UI-библиотек (Vuetify, Element Plus и т.д.) — проект использует чистый Tailwind.
